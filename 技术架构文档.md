# VoyageAI - 智能旅行规划系统技术架构文档

## 1. 系统架构概述

VoyageAI 采用前后端分离的架构设计，前端使用 Vue 3 构建现代化的单页应用，后端使用 FastAPI 提供高性能的 RESTful API 服务。系统集成了 AI 行程生成、实时天气查询等核心功能，通过外部 API 提供智能服务。

### 1.1 架构图

```
┌─────────────────┐
│   前端应用层    │
│ Vue 3 + Vite   │
│   (浏览器)     │
└─────────┬───────┘
          │ HTTP/JSON
┌─────────▼───────┐
│   API 网关层    │
│   FastAPI       │
└─────────┬───────┘
          │
┌─────────▼───────┐
│   业务逻辑层    │
│ AI行程生成      │
│ 天气数据处理    │
└─────────┬───────┘
          │
┌─────────▼───────┐
│   外部服务层    │
│ DeepSeek API    │
│ 高德地图API     │
└─────────────────┘
```

### 4.3 核心业务逻辑

#### 4.3.1 行程生成流程

1. 接收用户请求参数（出发地、目的地、天数、预算）
2. 构建提示词（Prompt）
3. 调用 DeepSeek API 生成行程内容
4. 解析和格式化返回数据
5. 返回结构化的行程数据

#### 4.3.2 天气查询流程

1. 接收城市名称参数
2. 调用高德地图地理编码API获取城市adcode
3. 使用adcode调用天气API获取实时天气
4. 生成天气预警和建议
5. 返回天气数据和预警信息

## 5. 数据库设计

### 5.1 数据库表结构

#### 5.1.1 users（用户表）

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | INTEGER | PRIMARY KEY | 用户ID |
| username | VARCHAR(50) | UNIQUE | 用户名 |
| email | VARCHAR(100) | UNIQUE | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### 5.1.2 itineraries（行程表）

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | INTEGER | PRIMARY KEY | 行程ID |
| user_id | INTEGER | FOREIGN KEY | 用户ID |
| start_location | VARCHAR(100) | NOT NULL | 出发地 |
| destination | VARCHAR(100) | NOT NULL | 目的地 |
| days | INTEGER | NOT NULL | 天数 |
| budget | VARCHAR(50) | NOT NULL | 预算 |
| itinerary_data | TEXT | NOT NULL | 行程数据（JSON格式） |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 6. 部署架构

### 6.1 开发环境

- **前端**：`npm run dev` → http://localhost:5173
- **后端**：`python main.py` → http://localhost:8000

### 6.2 生产环境

#### 6.2.1 前端部署

1. 构建静态文件：`npm run build`
2. 部署到 Nginx 或 CDN
3. 配置域名和HTTPS

#### 6.2.2 后端部署

1. 使用 Docker 容器化部署
2. 使用 Gunicorn + Uvicorn 作为 WSGI/ASGI 服务器
3. 配置 Nginx 反向代理
4. 数据库使用 SQLite（轻量级部署）或 PostgreSQL（生产环境）

### 6.3 环境变量配置

| 环境变量 | 描述 | 默认值 |
|---------|------|--------|
| DEEPSEEK_API_KEY | DeepSeek API密钥 | 必填 |
| AMAP_API_KEY | 高德地图API密钥 | 必填 |
| DATABASE_URL | 数据库连接URL | sqlite:///voyageai.db |
| CACHE_TIMEOUT | 缓存超时时间（秒） | 300 |

## 7. 安全架构

### 7.1 API安全

- **API密钥管理**：使用环境变量存储，不在代码中硬编码
