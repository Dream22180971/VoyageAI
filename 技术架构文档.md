                     │

| 层级 | 技术 | 版本 | 选型理由 |

## 2. 前端架构

### 2.1 目录结构

```
frontend/src/
├── views/
│   ├── Home.vue          # 首页：行程表单 + 功能卡片 + 数据统计
│   ├── Inspiration.vue   # 灵感发现：热门目的地 + 主题旅行
│   ├── Guide.vue         # 目的地指南：城市攻略 + 搜索
│   ├── Community.vue     # 社区足迹：故事 + 排行 + 话题
│   ├── Result.vue        # 结果页：行程 + 天气 + 预算 + 清单
│   └── About.vue         # 关于我们
├── router/index.js       # 6 个路由配置
├── App.vue               # 根组件
├── main.js               # 入口
└── style.css             # 全局基础样式
```

### 2.2 路由设计

```javascript
routes = [
  { path: '/',           component: Home },         // 首页
  { path: '/inspiration', component: Inspiration },  // 灵感发现
  { path: '/guide',      component: Guide },         // 目的地指南
  { path: '/community',  component: Community },     // 社区足迹
  { path: '/result',     component: Result },        // 结果页
  { path: '/about',      component: About },         // 关于我们
]
```

### 2.3 主题系统

```
CSS 变量体系：
:root          → 浅色模式变量（主色 #10B981）
html.dark-mode → 深色模式变量（主色 #60a5fa）

切换机制：
1. 点击导航栏月亮/太阳按钮
2. 切换 html.dark-mode class
3. 保存到 localStorage('theme')
4. 页面加载时读取恢复
```

### 2.4 移动端适配

```
断点：768px
- nav-links 隐藏，显示 hamburger 按钮
- hamburger 3 线变 X 动画
- mobile-menu transition 展开/收起
- 所有卡片单列布局
- 页脚单列折叠
```

### 2.5 结果页数据流

```
首页表单 → Base64 编码 → URL query param `d`
                               ↓
Result.vue mounted → 解码 → 解析 JSON → 渲染
                               ↓
                    天气 API → 填充天气模块
                    Canvas   → 绘制预算图表
```

### 2.6 PDF 导出流程

```
点击「导出指南」
  → 临时切浅色模式
  → html2canvas 截图 .result-content（2x 分辨率）
  → 恢复深色模式
  → jsPDF 创建 A4 PDF
  → 自动分页（图片超出时添加新页）
  → 下载 `旅行计划_{目的地}.pdf`
```

## 3. 后端架构

### 3.1 目录结构

```
backend/
├── main.py           # 主应用：路由 + 业务逻辑 + DESTINATION_DATA
├── config.py         # 环境变量配置
├── database.py       # SQLite 连接
├── models.py         # SQLAlchemy 模型（User, Itinerary）
├── requirements.txt  # Python 依赖
└── .env              # API Keys（不入 Git）
```

### 3.2 API 设计

| 方法 | 路径 | 功能 | 输入 | 输出 |
|------|------|------|------|------|
| POST | `/api/plan` | AI 行程生成 | `{start, dest, days, budget}` | 完整行程 JSON |
| GET | `/api/weather?city=` | 实时天气 | 城市名 | 温度/天气/预警 |
| GET | `/api/health` | 健康检查 | - | `{status}` |
| GET | `/api/config` | 配置信息 | - | 配置 JSON |

### 3.3 行程生成策略

```
请求 /api/plan
  ↓
优先：DeepSeek API 生成（实时 AI）
  ↓ 失败/超时
降级：DESTINATION_DATA 模板匹配（5 城市定制）
  ↓ 未匹配
兜底：通用行程模板
```

### 3.4 目的地定制数据

```python
DESTINATION_DATA = {
    "北京": { best_season, pace, highlights, daily_plan[4天], packing },
    "上海": { ..., daily_plan[3天] },
    "成都": { ..., daily_plan[3天] },
    "西安": { ..., daily_plan[3天] },
    "杭州": { ..., daily_plan[3天] },
}
```

每个城市包含：真实景点名称、具体路线、美食推荐、时间安排、行李清单。

### 3.5 天气查询流程

```
GET /api/weather?city=北京
  → 高德地理编码 API（城市名 → adcode）
  → 高德天气 API（adcode → 实时天气）
  → 生成预警和建议
  → 返回结构化数据
```

## 4. 部署架构

### 4.1 服务器

- 阿里云 ECS，IP: `47.103.214.75`
- OS: Linux
- 项目路径: `/www/wwwroot/voyageai/voyageai/`
- 管理面板: 宝塔

### 4.2 Nginx 配置

```
server {
    listen 443 ssl;
    server_name 47.103.214.75;

    # HTTPS 证书（自签名）
    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # 前端静态文件
    location / {
        root /www/wwwroot/voyageai/voyageai/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 4.3 后端进程管理

```ini
# /etc/systemd/system/voyageai.service
[Unit]
Description=VoyageAI Backend
After=network.target

[Service]
WorkingDirectory=/www/wwwroot/voyageai/voyageai/backend
ExecStart=/usr/bin/python3 main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

### 4.4 部署流程

由于服务器无 GitHub 代理访问权限，采用 SCP 部署：

```
1. 本地 npm run build（验证构建）
2. scp 上传修改的 .vue 文件到服务器
3. ssh 远程 npm run build
4. ssh systemctl restart voyageai（如有后端变更）
```

## 6. 性能优化

| 优化点 | 方案 |
|--------|------|
| 首屏加载 | Vite 代码分割、CSS/JS 压缩 |
| 主题切换 | CSS Variables + class 切换，无重绘闪烁 |
| API 响应 | 内存缓存（timeout 300s） |
| PDF 导出 | 截图方式避免中文字体加载 |
| 移动端 | 768px 断点响应式，汉堡菜单减少 DOM |
| 结果页视觉 | 渐变色 + Emoji 替代外部图片，零请求 |

## 7. 扩展性

- **新增页面**：在 `views/` 添加组件，`router/index.js` 注册路由，导航栏/页脚添加链接
- **新增城市数据**：在 `DESTINATION_DATA` 字典添加条目，在 Result.vue 添加渐变色和 Emoji
- **替换 AI 模型**：修改 `main.py` 中的 AI 调用部分
- **数据库迁移**：SQLAlchemy 支持切换到 PostgreSQL/MySQL
